---
title: "Update User Profile"
api: "PATCH /v1/users/:userId"
description: "Update a user's profile information"
---

Update profile information for a user. Regular users can only update their own profile, while admins can update any user's profile.

## Authentication

This endpoint requires authentication via Bearer token.

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Path Parameters

<ParamField path="userId" type="string" required>
  The unique identifier of the user to update
</ParamField>

## Body Parameters

<ParamField body="name" type="string">
  User's full name
</ParamField>

<ParamField body="phone" type="string">
  User's phone number
</ParamField>

<ParamField body="city" type="string">
  User's city
</ParamField>

<ParamField body="dob" type="string">
  Date of birth (ISO 8601 format or date string)
</ParamField>

<ParamField body="gender" type="string">
  User's gender. Options: `male`, `female`, `prefer not to say`
</ParamField>

<ParamField body="battingStyle" type="string">
  Batting style. Options:
  - `Left-hand bat`
  - `Right-hand bat`
  - `none`
</ParamField>

<ParamField body="bowlingStyle" type="string">
  Bowling style. Options:
  - `Right-arm fast`
  - `Right-arm medium`
  - `Left-arm fast`
  - `Left-arm medium`
  - `Slow left-arm orthodox`
  - `Slow left-arm chinaman`
  - `Right-arm Off Break`
  - `Right-arm Leg Break`
  - `none`
</ParamField>

<ParamField body="playingRole" type="string">
  Playing role. Options:
  - `Top-order batter`
  - `Middle-order batter`
  - `Lower-order batter`
  - `Opening batter`
  - `Wicket-keeper batter`
  - `Wicket-keeper`
  - `Bowler`
  - `All-rounder`
  - `none`
</ParamField>

<ParamField body="profilePicture" type="string">
  URL to the user's profile picture
</ParamField>

<ParamField body="shortName" type="string">
  User's short name or nickname
</ParamField>

<ParamField body="fcmToken" type="string">
  Firebase Cloud Messaging token for push notifications
</ParamField>

## Error Responses

<ResponseField name="400 Bad Request">
  Invalid request data or duplicate email
  
  ```json
  {
    "code": 400,
    "message": "Email already taken"
  }
  ```
</ResponseField>

<ResponseField name="403 Forbidden">
  User does not have permission to update this profile
  
  ```json
  {
    "code": 403,
    "message": "Forbidden"
  }
  ```
</ResponseField>

<ResponseField name="404 Not Found">
  User not found
  
  ```json
  {
    "code": 404,
    "message": "User not found"
  }
  ```
</ResponseField>
