---
title: "Admin - Manage Matches"
api: "GET /v1/admin/teamMatches"
description: "Admin endpoint to list and manage matches"
---

Retrieve a paginated and filterable list of all matches in the system. This endpoint is restricted to admin users only.

## Authentication

This endpoint requires admin authentication via Bearer token.

```bash
Authorization: Bearer YOUR_ADMIN_ACCESS_TOKEN
```

## Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of matches per page
</ParamField>

<ParamField query="team_name" type="string">
  Filter by team name (partial match)
</ParamField>

<ParamField query="city" type="string">
  Filter by city where the match is played
</ParamField>

<ParamField query="ground" type="string">
  Filter by ground/venue name
</ParamField>

<ParamField query="status" type="string">
  Filter by match status. Options:
  - `scheduled`
  - `live`
  - `completed`
  - `abandoned`
</ParamField>

## Delete Match

Admins can delete a match using:

```bash
DELETE /v1/admin/matches/:matchId
```

### Path Parameters

<ParamField path="matchId" type="string" required>
  The unique identifier of the match to delete
</ParamField>

### Response

Returns `204 No Content` on successful deletion.

### Example

```bash cURL
curl --request DELETE \
  --url 'https://api.cricpr.com/v1/admin/matches/60a7f8c8d5f3e2001f8e4e8a' \
  --header 'Authorization: Bearer YOUR_ADMIN_ACCESS_TOKEN'
```

## Error Responses

<ResponseField name="401 Unauthorized">
  Missing or invalid authentication token
  
  ```json
  {
    "code": 401,
    "message": "Please authenticate"
  }
  ```
</ResponseField>

<ResponseField name="403 Forbidden">
  User is not an admin
  
  ```json
  {
    "code": 403,
    "message": "Admin access required"
  }
  ```
</ResponseField>

<ResponseField name="404 Not Found">
  Match not found
  
  ```json
  {
    "code": 404,
    "message": "Match not found"
  }
  ```
</ResponseField>
