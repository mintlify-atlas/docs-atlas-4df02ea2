This guide explains how to implement offline scoring with batch synchronization for the CricPR API.

## Overview

Offline sync enables uninterrupted scoring when internet connectivity is unreliable. The system supports:

1. Local storage of scoring operations
2. Automatic sync when connection is restored
3. Deduplication to prevent duplicate data
4. Conflict resolution for concurrent edits

## Architecture

The offline sync system uses a hybrid approach:

```
┌─────────────┐
│   Mobile    │
│    App      │
└──────┬──────┘
       │
       ├─── Online ────► REST APIs ────► MongoDB
       │
       └─── Offline ───► Local Queue ───► Batch Sync API ───► MongoDB
```

<Note>
Offline sync is particularly important for cricket scoring at venues with poor connectivity, such as remote cricket grounds.
</Note>

const toSync = [];
const serverClientIds = new Set(
  serverState.recentBalls.map(ball => ball.clientId).filter(id => id)
);

for (const op of localOperations) {
  // Skip if operation timestamp is before last sync
  if (lastSyncTimestamp && new Date(op.timestamp) <= new Date(lastSyncTimestamp)) {
    continue; // Already synced

  if (op.type === 'recordBall' && serverClientIds.has(op.clientId)) {

  toSync.push(op);

### Mark as synced

Update local queue to mark synced operations:

```javascript
const syncedIds = new Set(
  response.data.processed.map(p => p.clientId)
);

// Also mark skipped operations as synced
response.data.skipped?.forEach(skip => {
  if (skip.reason === 'duplicate' || skip.reason === 'already_synced') {
    syncedIds.add(skip.clientId);
  }
});

const updated = queue.map(op => 
  syncedIds.has(op.clientId) ? { ...op, synced: true } : op
);

await AsyncStorage.setItem('offline_queue', JSON.stringify(updated));
```

## Online → Offline → Online transition

The system handles seamless transitions when connectivity drops mid-scoring:

**Scenario:**
1. User scores **online** → Some balls go directly to server ✅
2. Network drops **mid-match** → User continues offline 📱
3. Operations queued locally while offline
4. Network returns → Sync queued operations without duplicates

**How deduplication works:**

From `~/workspace/source/docs/OFFLINE_SCORING_IMPLEMENTATION.md:287-395`:

```javascript
async function deduplicateOperations(matchId, operations, serverState, lastSyncTimestamp) {
  const toSync = [];
  const skipped = [];

  // Get existing ball clientIds from server
  const existingClientIds = new Set(
    serverState.recentBalls
      .map(ball => ball.clientId)
      .filter(id => id)
  );

  for (const op of operations) {
    // Check if operation is newer than last sync
    if (lastSyncTimestamp && new Date(op.timestamp) <= new Date(lastSyncTimestamp)) {
      skipped.push({
        clientId: op.clientId,
        reason: 'already_synced',
        operation: op.type
      });
      continue;
    }

    // Check for duplicates based on clientId
    if (op.type === 'recordBall') {
      if (existingClientIds.has(op.clientId)) {
        // Check if data matches
        const existingBall = serverState.recentBalls.find(
          b => b.clientId === op.clientId
        );
        
        if (existingBall) {
          const dataMatches = 
            existingBall.runScored === op.data.runScored &&
            existingBall.extras === op.data.extras;
          
          if (dataMatches) {
            skipped.push({
              clientId: op.clientId,
              reason: 'duplicate',
              operation: op.type
            });
            continue;
          } else {
            // Conflict detected
            skipped.push({
              clientId: op.clientId,
              reason: 'conflict',
              operation: op.type,
              serverData: existingBall,
              clientData: op.data
            });
            continue;
          }
        }
      }
    }

    toSync.push(op);
  }

  return { toSync, skipped };
}
```

<Warning>
Always generate unique `clientId` values for each operation. Use a combination of timestamp and random string: `client_${Date.now()}_${randomString}`
</Warning>

For complete innings recorded offline, use the innings batch sync endpoint:

```javascript
POST /v1/sync/innings/:inningsId/batch
```

**Request body:**

```json
{
  "balls": [
    {
      "ball_id": "1765848428284",
      "innings_id": "64a1234567890abcdef12370",
      "over_number": 1,
      "ball_sequence": 1,
      "runs": 4,
      "extras_type": null,
      "extras_runs": 0,
      "striker_id": "64a1234567890abcdef12350",
      "non_striker_id": "64a1234567890abcdef12351",
      "baller_id": "64a1234567890abcdef12360",
      "created_at": "2024-07-15T14:00:00Z"
    }
    // ... 500 more balls
  ]
}
```

**Response:**

```json
{
  "status": "success",
  "data": {
    "ballsInserted": 500,
    "ballsSkipped": 0,
    "statsRecalculated": true,
    "inningsId": "64a1234567890abcdef12370",
    "matchId": "64a1234567890abcdef12347"
  }

```

This endpoint performs full recalculation of:
- Over statistics
- Innings totals
- Player batting stats
- Player bowling stats
- Match scorecard

From `~/workspace/source/docs/OFFLINE_BATCH_SYNC_RECOMMENDATION.md:154-254`:

```javascript
async function syncInningsBalls(inningsId, balls) {
  // Start transaction
  const session = await mongoose.startSession();
  session.startTransaction();

  try {
    // Group balls by over_number to create/update overs
    const ballsByOver = groupBallsByOver(balls);
    
    // Get or create overs for each over_number
    const overMap = await getOrCreateOvers(inningsId, ballsByOver, session);
    
    // Delete existing balls for this innings (clean slate)
    await Ball.deleteMany(
      { overId: { $in: Array.from(overMap.values()).map(o => o._id) } },
      { session }
    );
    
    // Insert all balls in correct order
    const insertedBalls = await insertBallsInOrder(balls, overMap, session);
    
    // Update over statistics
    await updateOversStatistics(overMap.values(), session);
    
    // Update innings statistics
    await updateInningsStatistics(inningsId, session);
    
    // Recalculate player stats
    await updateInningsStats(inningsId);
    
    // Commit transaction
    await session.commitTransaction();
    
    return {
      ballsInserted: insertedBalls.length,
      statsRecalculated: true
    };
  } catch (error) {
    await session.abortTransaction();
    throw error;
  } finally {
    session.endSession();
  }
}
```

<Note>
Innings batch sync uses a "delete and recalculate" strategy to ensure data consistency. This guarantees correct statistics regardless of what happened during offline scoring.
</Note>

## Conflict resolution

When conflicts are detected (same operation with different data), the API returns conflict details:

```json
{
  "conflicts": [
    {
      "clientId": "client_123456_abc",
      "operation": "recordBall",
      "serverData": {
        "runScored": 4,
        "extras": 0
      },
      "clientData": {
        "runScored": 6,
        "extras": 0
      }
    }
  ]
}
```

**Resolution strategies:**

1. **Server wins** (recommended): Keep server data, discard client changes
2. **Last write wins**: Use the most recent timestamp
3. **Manual resolution**: Notify user to resolve manually

## Best practices

<AccordionGroup>

<Accordion title="Generate unique client IDs">

Always use a unique identifier for each operation:

```javascript
function generateClientId() {
  return `client_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`;
}
```

</Accordion>

<Accordion title="Track last sync timestamp">

Store the last successful sync time for each match:

```javascript
await AsyncStorage.setItem(
  `lastSyncTimestamp:${matchId}`,
  serverResponse.lastSyncTimestamp
);
```

</Accordion>

<Accordion title="Periodic backup sync">

Even during online scoring, periodically sync to backup data:

```javascript
// Sync every 5 minutes during innings
setInterval(() => {
  if (isOnline && unsyncedOperations.length > 0) {
    syncQueue();
  }
}, 5 * 60 * 1000);
```

</Accordion>

<Accordion title="Clean up synced operations">

Remove old synced operations to prevent storage bloat:

```javascript
// Keep last 100 synced operations for safety
const unsynced = queue.filter(op => !op.synced);
const synced = queue.filter(op => op.synced).slice(-100);
await AsyncStorage.setItem('offline_queue', JSON.stringify([...unsynced, ...synced]));
```

</Accordion>
